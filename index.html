Repo-backup script generator

If you need to back up all your repos somewhere, and you have a lot of them, cloning them one by one is a pain. And you can forget which repos you've cloned.

This page generates a shell script that clones all your repos in one go. If a repo already exists, it will just pull the latest changes from the default branch.

Usage

  1. Set yourself up to connect to GitHub with ssh automatically.
  2. To get your organisation's repos, enter its GitHub name below. To get your personal repos, select 'User' and enter your GitHub username.
  3. If you want to include private repos, you'll need a GitHub Personal API token with full repo scope. This site does nothing with your API token except use it in your browser directly to fetch your repo listing. (You can check the code yourself to make sure.) Only use this site over https for security, and delete your token on GitHub once you no longer need it.
  4. Click 'Generate'.
  5. To add another organisation's or user's repos to the same script, add their details to the form and click 'Generate' again.
  6. Copy and paste the text into a .sh file.
  7. Run that script where you store your repos. They will be cloned there in organisation folders.
